---
description: 新功能实现的标准工作流
---

1. **需求理解与上下文收集**:
   - 仔细阅读用户需求。
   - 使用 `find_by_name` 或 `grep_search` 查找相关的现有代码和目录结构。
   - 理解现有的架构模式 (Architecture Patterns)。

2. **制定实现计划**:
   - 规划需要创建或修改的文件。
   - 如果变更较大，创建一个简短的 Implementation Plan。

3. **代码实现**:
   - **基础建设**: 编写必要的类型定义、配置或数据库迁移。
   - **核心逻辑**: 实现核心功能函数或类。
   - **界面/API**: 实现前端组件或后端接口。
   - *注意*: 遵循现有的代码风格和规范。

4. **验证与测试**:
   - 编译/构建项目检查类型错误 (`npm run build` 或类似)。
   - 运行相关测试或手动验证功能。

5. **代码完善**:
   - 优化代码结构。
   - 添加必要的注释。
   - 更新相关文档 (如有需要)。
