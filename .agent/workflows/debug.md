---
description: 系统化调试流程技能
---

1. **问题分析**:
   - 阅读报错信息或用户描述的异常行为。
   - 如果有堆栈跟踪 (Stack Trace)，优先根据堆栈定位文件和行号。
   - 使用 `grep_search` 查找相关错误关键字。

2. **复现与验证 (若适用)**:
   - 检查是否有现成的测试用例失败。
   - 如果没有，尝试编写一个复现脚本或测试用例。
   - 运行相关命令确认问题复现。

3. **定位根本原因**:
   - 阅读相关代码逻辑。
   - 检查数据流和状态变化。
   - 使用 `view_file` 深入查看上下文。

4. **实施修复**:
   - 修改代码以修复 Bug。
   - 确保修改的副作用最小。

5. **验证修复**:
   - 再次运行复现脚本或测试用例，确保通过。
   - 检查是否引入了新的问题。

6. **总结**:
   - 解释问题原因和修复方案。
